# RAG + Agent System Design Template

## 1. Problem

What user problem does the system solve?

## 2. Users

Who uses it?

## 3. Data Sources

What documents, APIs, databases, or tools does it need?

## 4. Core Workflow

Describe the request path:

1. user request
2. auth
3. safety checks
4. retrieval or routing
5. model/tool call
6. output guardrail
7. response

## 5. Architecture Diagram

Add a Mermaid diagram.

## 6. Reliability

- What can fail?
- What is cached?
- What uses a queue?
- What fallback exists?

## 7. Security

- What data is sensitive?
- What tools are risky?
- Where is human approval needed?

## 8. Observability

- What logs are captured?
- What metrics matter?
- What traces are needed?

## 9. Evaluation

- What golden cases exist?
- What metrics define success?

## 10. Cost

- What are the costly steps?
- How can cost be reduced?
