# Start Here: Your AI Transition Path

This is the single entry point for the roadmap.

If you are confused by many folders, use this rule:

> Study `M0` to `M23` in order. Everything else is support material.

## What To Open First

Open files in this order:

1. `00-Start-Here/README.md` — this file.
2. `AI-Master-Roadmap.md` — week-by-week curriculum.
3. `M0-AI-Engineering-Foundations/README.md` — your first module.
4. `99-End-to-End-Practice/README.md` — labs after you start building.

Do not begin with `COMPREHENSIVE-THEORY-GUIDE.md`. Use it when a module feels unclear and you want deeper explanation.

## Daily Study Loop

For every module:

1. Read the module `README.md`.
2. Write a 5-line summary in your own words.
3. Run one code example.
4. Break it intentionally.
5. Fix it.
6. Complete the beginner exercise.
7. Complete one advanced exercise.
8. Update your notes.

## Weekly Review Loop

At the end of each week:

- review notes
- rerun code examples
- answer interview questions
- update project README
- write what still feels confusing

## Beginner Mindset

You are transitioning into AI engineering. You are not expected to know everything before starting.

First pass goal:

- recognize the concepts
- run the examples
- understand where each piece fits

Second pass goal:

- modify examples
- connect modules
- build projects

Third pass goal:

- explain trade-offs
- add tests/evals/security
- make portfolio-ready work

## Main Skill Map

| Stage | Skill | Beginner meaning |
| --- | --- | --- |
| Foundation | Python/API/LLM | call models and validate outputs |
| RAG | Embeddings/retrieval | answer from documents |
| Agents | Tools/workflows | let AI take structured actions |
| Production | Security/observability/cost | make AI safe and debuggable |
| Enterprise | Cloud/product/governance | make AI useful in companies |
| Capstone | Integration | combine everything |
