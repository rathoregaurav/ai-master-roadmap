## Week 23: Repair Week

Use this week to revisit weak spots.

### Checklist

- [ ] Can I run Phase 1 examples?
- [ ] Can I explain RAG from ingestion to answer?
- [ ] Can I explain an agent loop?
- [ ] Can I explain tool calling?
- [ ] Can I explain prompt injection?
- [ ] Can I explain observability fields?
- [ ] Can I explain AWS and Kubernetes at a beginner level?

### Practice

Pick the three weakest areas and write:

1. What confused me?
2. What is the simple explanation?
3. What code example proves I understand it?
4. What interview question could test it?

## Week 24: System Design Week

Design your enterprise AI platform on paper.

## Final Design Prompt

Design an enterprise AI assistant that:

- lets users ask questions over private documents
- supports citations
- uses agents for tool workflows
- has prompt injection checks
- tracks cost and latency
- stores audit logs
- supports multiple tenants
- deploys as containers
- has a feedback system

## Deliverables

- architecture diagram
- API list
- data model
- AWS resource map
- Kubernetes deployment plan
- observability plan
- security plan
- evaluation plan
- cost plan

## Interview Practice Questions

1. How does a document move from upload to retrieval?
2. How do you prevent tenant data leakage?
3. What happens if the model provider is down?
4. How do you evaluate answer quality?
5. How do you reduce cost?
6. How do you roll back a bad prompt?
7. What should appear in an audit log?
