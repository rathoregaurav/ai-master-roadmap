# Lab 5: Enterprise AI Platform Layer

## Goal

Add cloud, product, and enterprise governance concepts to your AI system.

## Beginner Version

Create:

- AWS resource map for your RAG app
- Kubernetes deployment and service
- feedback event model
- prompt registry entry
- model registry entry
- audit log event
- lineage record

## Advanced Version

Add:

- tenant-aware access model
- feature flags
- A/B prompt experiment
- deployment rollback plan
- admin governance checklist
- system design diagram

## Deliverable

A folder or README containing:

- enterprise architecture diagram
- resource map
- deployment manifest
- product feedback schema
- governance checklist
- audit and lineage examples

## Reflection Questions

1. What data needs tenant isolation?
2. Which model and prompt versions need approval?
3. What should be visible to admins?
4. What events must be audited?
5. What would fail first at enterprise scale?
