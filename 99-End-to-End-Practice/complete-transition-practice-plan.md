# Complete AI Transition Practice Plan

## Purpose

This plan connects beginner learning to job-ready AI engineering practice.

## Month 1: Foundations And LLM Apps

Build:

- typed LLM wrapper
- structured output prompt
- FastAPI AI endpoint
- streaming endpoint

You should be able to explain:

- tokens
- context windows
- system prompts
- Pydantic validation
- streaming responses

## Month 2: RAG

Build:

- notes RAG CLI
- document chunker
- in-memory vector store
- retrieval evaluation
- RAG answer system with citations

You should be able to explain:

- embeddings
- cosine similarity
- chunking trade-offs
- hybrid search
- Recall@K and MRR

## Month 3: Agents

Build:

- tool registry
- beginner agent loop
- router and supervisor patterns
- MCP-style connector
- memory store
- agent evaluation harness

You should be able to explain:

- tool calling
- ReAct
- agent state
- MCP tools and resources
- memory types
- agent evaluation

## Month 4: Production AI Engineering

Build:

- request trace logger
- prompt injection detector
- PII scrubber
- circuit breaker
- latency benchmark
- model router
- production readiness checklist

You should be able to explain:

- logs, metrics, and traces
- prompt injection
- tool permissions
- caching and queues
- p50, p95, and p99 latency
- model routing and cost estimation

## Month 5: Infrastructure And Enterprise AI

Build:

- AWS resource map
- Kubernetes deployment manifest
- AI chat product feedback schema
- prompt registry
- model registry
- audit log
- data lineage record
- tenant-aware control plane

You should be able to explain:

- S3, Lambda, ECS, IAM, and Secrets Manager
- deployments, services, and ingress
- streaming chat UX and citations
- feature flags and A/B tests
- prompt and model governance
- audit logs, data lineage, and multi-tenancy

## Portfolio Rule

Every month must end with:

- one working demo
- one architecture diagram
- one README
- one evaluation report
- one reflection note
