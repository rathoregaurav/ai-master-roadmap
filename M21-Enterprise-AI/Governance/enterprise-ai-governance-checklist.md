# Enterprise AI Governance Checklist

## Prompt Governance

- [ ] Prompt templates have names and versions.
- [ ] Prompt owners are assigned.
- [ ] Prompt changes are reviewed.
- [ ] Prompt versions are logged per request.

## Model Governance

- [ ] Approved model list exists.
- [ ] Model cost and risk are documented.
- [ ] Fallback models are defined.
- [ ] Model usage is logged.

## Data Governance

- [ ] Data sources are registered.
- [ ] Tenant boundaries are enforced.
- [ ] Data lineage is recorded.
- [ ] Retention policy is documented.

## Audit Governance

- [ ] User requests are auditable.
- [ ] Tool calls are auditable.
- [ ] Admin changes are auditable.
- [ ] Security flags are auditable.

## Product Governance

- [ ] Feedback is collected.
- [ ] Experiments are tracked.
- [ ] Feature flags are controlled.
- [ ] Rollback paths exist.
