# M16 Practice: AWS for AI

## Beginner Exercises

1. Read the AWS RAG resource map.
2. Draw a document upload to RAG indexing workflow.
3. Run `iam_policy_builder.py`.
4. Explain why the policy is read-only.
5. List which secrets your AI app would need.

## Advanced Exercises

1. Design S3 bucket prefixes for raw, processed, and archived documents.
2. Write an IAM policy for write access to only one prefix.
3. Design an SQS queue for ingestion jobs.
4. Write a failure handling plan for document parsing.
5. Add CloudWatch log fields you would track.

## Done When

- [ ] You can explain S3, Lambda, ECS, IAM, Secrets Manager, and VPC.
- [ ] You can design async document ingestion.
- [ ] You can explain least privilege.
- [ ] You can describe how a RAG app runs in AWS.
